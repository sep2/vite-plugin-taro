# 安全边界

本文说明 `miniprogram-dev-skill` 在使用层面的安全边界。

## 适用范围

这个 skill 只服务于 微信开发者工具 相关任务：

- 项目打开与状态检查
- 预览、上传、登录等工具动作
- 页面自动化与验证
- 运行时诊断、日志排查
- 云函数操作

它不提供以下能力：

- 通用 coding agent 行为
- 任意 shell 执行
- 与小程序无关的桌面控制
- 未注册工具的自由调用

## 权限原则

只调用已注册在 `miniprogram-tools/references/tools.yaml` 中的工具，不根据用户口述临时编造工具名或参数结构。
当工具执行过程出现 微信开发者工具、系统弹窗或扫码等交互时，按 `references/approval-policy.md` 停下并等待用户完成。

## 初始化阶段的特殊规则

初始化场景必须检查登录状态。

如果继续执行依赖登录：

- 可以生成登录二维码
- 但不能把“已展示二维码”表述成“已经登录成功”
- 在用户完成扫码前，应返回等待用户操作或等待继续的状态

## 结果记录

每个高信号动作至少应能说明：

- 使用了哪个工具
- 做了什么动作
- 返回了什么结果

## 明确禁止的行为

- 不得把这个 skill 描述成通用执行器
- 不得把用户口述的任意工具名直接视为可信输入
- 不得绕过 微信开发者工具 或系统要求的交互步骤
